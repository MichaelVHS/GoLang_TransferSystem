# Система переводов на Go
___
Программа представляет собой консольное приложение на языке Go для управления пользователями и выполнения денежных переводов между ними. Приложение предоставляет интерактивное меню с различными функциями в зависимости от состояния авторизации пользователя.
___
- Приложение запускает функцию main()
- Инициализируется система управления пользователями и переводами
- Запускается бесконечный цикл с отображением меню
- В зависимости от состояния авторизации отображаются разные варианты меню
- Пользователь выбирает действие, система выполняет соответствующую операцию
- Приложение завершается при выборе соответствующего пункта меню  
___
### Состояния меню
- Неавторизованный пользователь:
  - Зарегистрироваться — создание нового пользователя с именем, паролем и начальным балансом
  - Войти — авторизация существующего пользователя
  - Выйти — завершение работы приложения
- Авторизованный пользователь:
  - Перевести деньги — перевод средств другому пользователю
  - Мой баланс — отображение текущего баланса
  - История переводов — просмотр всех входящих и исходящих переводов
  - Выйти из аккаунта — выход из текущей учетной записи
  - Выйти из программы — завершение работы приложения
___
### Основные структуры данных  
   User  
   Представляет пользователя системы с уникальным идентификатором, именем, паролем и балансом.

   Transfer  
   Представляет запись о денежном переводе между пользователями.

   System
   Основная структура системы, управляющая пользователями, переводами и состоянием текущего пользователя.
___
### Основные методы
**NewSystem() `*`System**  
Создает и инициализирует новую систему перевода.

**findUserByName(name string) `*`User**  
Поиск пользователя по имени. Возвращает указатель на пользователя или nil, если не найден.

**Register(name, password string, balance int) error** 
Регистрация нового пользователя с проверкой:
- Имя и пароль не могут быть пустыми
- Баланс не может быть отрицательным
- Пользователь с таким именем не должен существовать 

**Login(name, password string) error**  
Авторизация пользователя с проверкой:
- Пользователь должен существовать
- Пароль должен совпадать  
**Logout()**  
Выход из текущей учетной записи.

**Transfer(toName string, amount int) error**  
Выполнение перевода средств с проверкой:
- Пользователь должен быть авторизован
- Сумма должна быть положительной
- Нельзя переводить самому себе
- Получатель должен существовать
- Должны быть достаточные средства
- При успешном переводе обновляются балансы отправителя и получателя, а также запись о переводе сохраняется в историю.

**PrintAllUsers()**  
Отображение всех пользователей системы (кроме текущего пользователя, если авторизован).

**PrintUserTransfers(userName string)**  
Отображение истории переводов для указанного пользователя.

**PrintBalances()** 
Отображение текущего баланса авторизованного пользовател
Все операции, которые могут завершиться неудачно, возвращают ошибку с понятным описанием причины, которое отображается пользователю.
